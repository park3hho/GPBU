# 6주차 1교시: 설계의 개념과 유형 및 소프트웨어 아키텍처
## U21. 설계의 개념과 유형
### (1) 설계의 개념
요구사항 분석 단계에서 규명된 필수 기능을 어떻게 구현할 수 있는가에 대한 방법을 명시하는 것

**원칙**
- 소프트웨어 설계는 변경이 용이하도록 구조화
- 한 함수 안에 특정 기능을 수행하는 데 필요한 자료만을 사용하도록 규제
- 제작되는 과정에서 품질이 평가
- 계층적 조직이 제시되며, 모듈적


### (2) 설계의 유형
기술적 시각
#### ㄱ. 데이터 설계
- 소프트웨어 구조 설계와 연관되어 있음
- 자세한 자료 설계는 소프트웨어 컴포넌트가 설계될 때 가능
#### ㄴ. 구조 설계
- 집의 평면도와 같이 소프트웨어 전체를 전망할 수 있도록 해줌
- 소프트웨어의 주요 구성요소, 시스템 요구사항을 만족시키는데 사용될 수 있는 설계 패턴과 구조 양식 그리고 구조를 구현하는데 영향을 미치는 제약사항을 정의함.
#### ㄷ. 인터페이스 설계
- 소프트웨어가 상호작용하는 시스템, 그리고 그 소프트웨어를 사용하는 인간과 어떻게 대화하는지를 나타내는 설계를 의미
#### ㄹ. 컴포넌트 수준 설계
- 각 소프트웨어 컴포넌트의 자세한 내부 내용을 기술
#### ㅁ. 배치 수준 설계
- 소프트웨어를 지원하는 물리적 컴퓨팅 환경 내에서 소프트웨어 기능과 서브시스템을 어떻게 할당할지 나타냄

> 기본 설계
> 전체적인 하드웨어구조, 소프트웨어구조, 제어구조, 데이터구조, 사용자지침서, 검사 계획서 작성
