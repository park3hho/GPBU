# 5주차 1교시: 유용한 알고리즘 _ 탐색
## U21. 탐색 알고리즘 개요
탐색 알고리즘의 두가지 방법
- 탐색 알고리즘
- **해*** 탐색 알고리즘
  - 인공지능에서 사용하는 용어

탐색 알고리즘의 개념
- 탐색 뿐만 아니라 데이터의 삽입, 삭제 연산 기능에도 사용됨. (어디 넣을지 무엇을 삭제할지 찾아야 하니)

탐색 알고리즘의 종류
- 순차 탐색
- 이진 탐색
  - 배열
  - 링크드 리스트
- 이진 탐색 트리
- 해시 탐색

## V22. 순차 탐색 알고리즘과 이진 탐색 알고리즘
### (1) 순차 탐색 알고리즘
**정의**: 선형 탐색이라 불리는 순차탐색은 가장 간단한 탐색 방법  
**시간 복잡도**: $O(n)$

### (2) 이진 탐색 알고리즘
**정의**: 정렬이 이루어 지지 않을 때 사용하면 좋은 방법  
**시간 복잡도**: $O(log{n})$ 
- 삽입과 삭제 시 느린 단점이 있음

## W23. 이진 탐색 트리 알고리즘과 해시 탐색 알고리즘
### (1) 이진 탐색 트리 알고리즘
**정의**: BST라고 불림, 이진 탐색의 단점을 보완하기 위해 나옴  
**시간 복잡도**: $O(log{n})$ 

### (2) 해시 탐색 알고리즘
**정의**: 저장부터 해시함수를 이용하고, 해시 함수를 이용하여 탐색하는 방법 (데이터의 값과 인덱스를 미리 연결시킴)  
**시간 복잡도**: $O(1)$

# 5주차 2교시: 유용한 알고리즘 _ 그래프
## W23. 그래프 알고리즘의 개요
한붓 그리기 규칙은 연결되어 있는 선이 **홀수개**인, 점이 없거나 두개만 있는 것에서만 가능함

### (1) 그래프의 정의
정점과 간선들의 모음
### (2) 그래프의 용어
- **인접 (adjacent)** : 간선으로 연결되어 있는 두 정점, 이웃관계에 있다고 말할 수 있음
- **경로 (path)** : 간선을 통해 서로 이웃된 각 정점들로 그래프 안에서 형성된 길을 말함
- **차수 (degree)** : 정점에 속한 간선들의 수를 말함

**방향 그래프**: 간선에 방향성이 있는 그래프  
**무방향 그래프**: 간선에 방향성이 없는 그래프  

**연결 (connected)**: 무방향 그래프 내에서, 그래프 내에 모든 정점이 연결되어 있으면. `연결`  
**사이클 (cycle)**: 어느 경로가 정점 하나를 두번 거치도록 되어있는 경로.

### (3) 그래프의 표현
#### ㄱ. 인접 리스트
- 리스트로 표현
- 무방향 그래프일 경우 헤드 노드 뒤에 연결된 노드의 개수가 그 노드의 차수임
#### ㄴ. 인접 행렬
- 행렬로 표현
- 인접 리스트보다 빠름

## X24. 그래프 알고리즘의 2가지 순회방법
**정의**: 그래프의 모든 정점을 순차적으로 방문하는 것

- 자료구조상 탐색 방법 / 자료 구조를 그래프로 표현했을 때 탐색하는 방법이라는 의미임
  - 깊이 우선 탐색
  - 너비 우선 탐색

### (1) 깊이 우선 탐색
**동작**: 분기별 모든 정점 다 찾을 때까지 동작

재밌구만, 동작하는거. 표로 보면.

### (2) 너비 우선 탐색
**동작**: 인접한 정점을 모~~~~두 방문해버림.
