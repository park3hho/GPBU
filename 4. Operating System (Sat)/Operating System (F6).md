# 6주차 2교시: 공유자원 및 임계구역 접근방법



운영체제에서 공유자원(Shared Resource) 과 임계구역(Critical Section) 문제는 다중 프로세스 또는 스레드가 동시에 실행될 때 반드시 해결해야 하는 핵심적인 동기화 문제입니다.

1. 공유자원 및 임계구역 문제 개요

공유자원이란 둘 이상의 프로세스 또는 스레드가 동시에 접근할 수 있는 자원을 의미하며, 대표적으로 메모리, 파일, 변수 등이 이에 해당합니다. 이러한 공유자원에 여러 프로세스가 동시에 접근하게 되면 데이터의 일관성이 깨지거나 예상하지 못한 결과가 발생하는 문제가 생길 수 있습니다. 이때 하나의 프로세스만 접근할 수 있도록 제한해야 하는 코드 영역을 임계구역이라고 하며, 임계구역 문제는 여러 프로세스가 동시에 해당 영역에 진입하려 할 때 발생하는 동기화 문제를 의미합니다.

2. 임계구역 해결을 위한 조건

임계구역 문제를 해결하기 위해서는 다음의 세 가지 조건을 만족해야 합니다.

상호 배제(Mutual Exclusion)
하나의 프로세스가 임계구역에 진입해 있을 경우 다른 프로세스는 접근할 수 없어야 합니다.

진행(Progress)
임계구역에 아무도 없을 때, 진입하려는 프로세스가 있다면 그 중 하나는 반드시 들어갈 수 있어야 합니다.

한정 대기(Bounded Waiting)
특정 프로세스가 무한정 대기하지 않도록 진입 기회가 제한된 횟수 내에 주어져야 합니다.

3. 해결 방법
3.1 소프트웨어적 해결 방법

피터슨 알고리즘 (Peterson’s Algorithm)
두 개의 프로세스가 공유자원을 사용할 때 상호 배제를 보장하는 알고리즘으로, turn 변수와 flag 변수를 이용하여 임계구역 진입 여부를 제어합니다.

3.2 하드웨어적 해결 방법

Test and Set 명령어
임계구역에 진입하기 전에 lock 값을 검사하고 설정하는 과정을 하나의 원자적 연산으로 수행하여 다른 프로세스의 접근을 방지합니다.

Compare and Swap (CAS)
특정 메모리 값이 예상한 값과 동일할 경우에만 새로운 값으로 변경하는 방식으로 동기화를 구현합니다.

3.3 동기화 도구를 이용한 해결 방법

Mutex Lock
하나의 스레드만 임계구역에 접근할 수 있도록 잠금 기능을 제공합니다.

Semaphore
공유자원에 접근할 수 있는 프로세스의 개수를 제한하여 동기화를 구현하는 방법입니다.

Monitor
공유자원에 접근하는 코드를 하나의 모듈로 구성하여 자동으로 상호 배제를 보장하는 고급 동기화 기법입니다.

4. 결론

공유자원과 임계구역 문제는 다중 프로세스 환경에서 필연적으로 발생하는 문제이며, 이를 해결하기 위해 다양한 소프트웨어적 및 하드웨어적 기법이 사용됩니다. 이러한 해결 방법을 통해 프로세스 간의 충돌을 방지하고 시스템의 안정성과 데이터의 일관성을 유지할 수 있으며, 효율적인 자원 관리를 가능하게 합니다.
