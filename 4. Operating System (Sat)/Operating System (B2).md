# 2주차 2교시: 컴퓨터 
## G7. 컴퓨터의 성능을 향상시키는 기술
### (1) 버퍼
정의: 속도에 차이가 있는 두 장치 사이에서 그 차이를 완화하는 역할을 하는 장치
- 일정량의 데이터를 모아 옮김으로 속도의 차이를 완화


#### ㄱ. 스풀
**정의**: CPU와 입출력장치가 독립적으로 동작하도록 고안된 **소프트웨어적인 버퍼**
- 인쇄할 내용을 순차적으로 출력하는 소프트웨어로 출력
- 인쇄물이 완료될 때까지 다른 인쇄물이 끼어들 수 없으므로 프로그램 간에 배타적임

#### ㄴ. 캐시
**정의**: 메모리와 CPU 간의 속도 차이(BSB&rlarr;FSB)를 완화하기 위해 메모리의 데이터를 미리 가져와 저장해도는 임시 장소
- 필요한 데이터를 모아 한꺼번에 전달하는 버퍼의 일종으로 CPU가 사용할 것으로 예산되는 데이터를 미리 가져다놓음.
- CPU는 메모리에 접근해야 할 때 캐시를 먼저 방문하여 원하는 데이터가 있는지 찾아봄.

**즉시쓰기**(Write Through)
- 캐시에 있는 데이터가 변경되면 이를 즉시 메모리에 반영하는 방식
- 메모리와의 빈번한 데이터 전송으로 인해 성능이 느려짐
- 메모리의 최신값이 유지되기 때문에 정전에도 데이터를 잃지 않음

**지연쓰기**(Write back)
- 캐시에 있는 데이터가 변경되면 이를 즉시 메모리에 반영하는 것이 아니라 변경된 내용을 주기적으로 반영하는 방식
- 카피 백이라고도 함
- 메모리와의 데이터 전송 횟수가 줄어들어 시스템의 성능을 향상시킬 수 있다.
- 메모리와 캐시 데이터 간의 불일치가 생길 수 있음

### (2) 인터럽트
#### ㄱ. 폴링(Polling)
**정의**: CPU가 직접 입출력장치에서 데이터를 가져오거나 내보내는 방식
- CPU가 입출력장치의 상태를 주기적으로 검사하여 일정한 조건을 만족할 때 데이터를 처리
- CPU가 명령어 해석과 실행이라는 본래 역할 외에 모든 입출력까지 관여해야 하므로 작업 효율이 떨어짐.

#### ㄴ. 인터럽트(Interrupt)
**정의**: 입출력 관리자가 대신 입출력을 해주는 방식
- CPU의 작업과 저장장치의 데이터 이동을 독립적으로 운영함으로써 시스템의 효율을 높임.
- 데이터의 입출력이 이루어지는 동안 CPU가 다른 작업을 할 수 있음.

**용어**
인터럽트: 
- 입출력 관리자가 CPU에 보내는 완료 신호    

인터럽트 번호:
- 많은 주변장치 중 어떤 것의 작업이 끝났는지 CPU에 알려주기 위해 사용하는 번호
- 윈도우 운영체제에서는 IRQ라 부름.

인터럽트 벡터:
- 여러개의 입출력 작업을 한꺼번에 \처리하기 위해 여러 인터럽트를 하나의 배열로 만든 것.


**직접 메모리 접근 DMA**
**정의**: 입출력 관리자가 CPU의 허락 없이 메모리에 접근할 수 있는 권한
- 메모리는 CPU의 작업 공간이지만, 데이터 전송을 지시 받은 입출력 관리자는 직접 메모리 접근 권한이 있어야만 작업을 처리할 수 있음.

**메모리 매핑 입출력**
**정의**: 메모리의 일정 공간을 입출력에 할당하는 기법

**사이클 훔치기**
**정의**: CPU의 작업 속도보다 입출력장치의 속도가 느리기 때문에 직접 메모리 접근에 양보하는 것

## H8. 병렬처리 개념
### (1) 병렬 처리 개념
**정의**: 동시에 여러 개의 명령을 처리하여 작업의 능률을 올리는 방식

**병렬 처리 순서**  
명령어 패치(IF)  
명령어 해석 (ID)  
실행  
쓰기: 메모리에 저장

- 네 단계로 나누어 각 블록을 하나만 실행되게 해서 병렬 처리함. (나머지는 겹쳐도 됨)

#### ㄱ. 파이프라인 기법
- CPU의 사용을 극대화하기 위해 명령을 겹쳐서 사용하는 기법
- 하나의 코어에 멀티 스레드를 이용하는 방식

데이터 위험
- 데이터 의존성으로 인해 발생하는 문제임
- 데이터 A를 필요하는 두번째 명령엉가 앞의 명령어가 끝날 때까지 동시에 실행되어선 안됨

제어 위험
- 프로그램 카운터 값을 갑자기 변화시켜 발생하는 위험
- 첫 명령어를 실행하고 보니 goto문이어서 다음 문장이 아니라 다른 문장으로 이동하게 되면 현재 동시 처리되고 있는 명령어는 의미가 없어짐
- 분기 예측이나 분기 지연방법으로 해결

구조 위험
- 서로 다른 명령어가 같은 자원에 접근할 때 생기는 위험

#### ㄴ. 슈퍼스칼라 기법
- 듀얼코어 CPU를 이용해 2개의 작업을 동시에 처리하는 방식
- 파이프라인을 처리할 수 있는 코어를 여러 개 구성하여 복수의 명령어가 동시에 실행되도록 하는 방식

#### ㄷ. 슈퍼파이프라인 기법
- 파이프라인의 각 단계를 세분하여 한 클록 내에 여러 명령어를 처리

#### ㄹ. 슈퍼파이프라인 슈퍼스칼라 기법
- 슈퍼파이프라인 기법을 여러 개의 코어에서 실행

#### ㅁ. VLIW(Very Long Instruction Word)
- CPU가 병렬 처리를 지원하지 않을 경우 소프트웨어적으로 병렬처리하는 방법
- 동시에 수행할 수 있는 명령어들을 컴파일러가 추출하고 하나의 명령어로 압축하여 실행

### (2) 병럴 처리시 고려 사항
- 상호 의존성이 없어야 병렬 처리가 가능
- 각 단계의 시간을 일정하게 맞춰야함
