# 1주차 1교시: OS 개요
## A1. 운영체제의 발전과정
### (1) 운영체제의 종류
#### ㄱ. 임베디드 운영체제 
- CPU의 성능이 낮고 메모리 크기도 작은 시스템에 내장하도록 만든 우영체제,
- 임베디드 운영체제의 기계는 기능을 계속 향상시킬 수 있음.

### (2) 운영체제의 역사
**1940년대**: 운영체제 없음 (초기 전자식 디지털 컴퓨터)  
**1950년대**: 한 번에 하나의 일만 처리 (단순 순차처리)  
**1960년대**: 멀티 프로그래밍, 시분할, 멀티 프로세싱 개념 대두 및 보편화 (다중 프로그래밍)  
**1970년대**: 일괄처리: 시분할 처리, 실시간 처리 방식의 보편화 (다중모드 시분할)  
**1980년대**: 네트워크를 기반으로 클라이언트-서버 모델 확대 (분산 네트워크)  
**1990년대**: PC용 운영체제와 서버용 운영체제의 분리 및 보편화 (병렬계산, 분산계산)  
**2000년대**: 가전체품 및 모바일 장치를 위한 소형 운영체제 보편화 (모바일 및 임베디드)  

**1940년대**: [진공관] 운영체제가 없음.  
**1950년대**: [천공카드] 메인 메모리가 운영체제의 상주 영역과 사용자의 영역으로 나뉨.   
**1960년대**: [키보드와 마우스] 시분할을 이용할 수 있게 됨.  
**1970년대**: [분산 시스템] 개인용 컴퓨터의 보급, 이에 따른 운영체제의 발전  
**1990년대**: [클라이언트 서버 시스템] 작업을 요청하는 클라이언트와 이에 응답하여 요청을 작업하는 서버의 이중 구조로 나뉨. 근데 이게 운영체제랑 무슨 상관인지는 이야기 안함.  
**2000년대**: [P2P 시스템] 클라이언트-서버 구조의 단점인 과부화를 피하기 위해 만듦.

#### ㄱ. 그리드 컴퓨팅
- 정의: 흩어져 있는 컴퓨터의 남는 자원을 하나로 묶어 계산할 수 있게 만드는 방식

#### ㄴ. 클라우드 컴퓨팅
- 정의: 어디에서나 응용프로그램과 데이터를 자유롭게 사용할 수 있는 컴퓨팅 방식

#### ㄷ. 사물 인터넷 IoT
- 그냥 사물들에 인터넷이 들어가버린 상황.

## B2. 운영체제의 정의 및 역할
- 운영체제: 응용 프로그램, 사용자에게 컴퓨터 자원을 사용할 수 있는 인터페이스를 제공하고 그 결과를 돌려주는 시스템 소프트웨어

### (3) 운영체제의 역할
#### ㄱ. 프로세스 관리
- 프로세스는 일반적으로 수행 중인 프로그램을 의미함.
- 수행 중은 프로그램이 컴퓨터 시스템에 입력되어 운영체제에 등록되어 있음을 의미함.
- 프로세스는 실행 상태, 실행 준비 상태, 대기 상태로 구분됨.
#### ㄴ. 프로세스 관리
- 각 프로세스들을 어느 프로세서에 배당하여 처리하도록 할 것인지를 결정하며, Scheduler가 담당함.
#### ㄷ. 입출력 장치 관리
#### ㄹ. 기억장치 관리
- 보조기억장치로부터 주기억장치에 적재하는 역할
#### ㅁ. 자원 관리
소프트웨어 관리와 하드웨어 관리로 나뉨.
### (4) 운영체제의 목표
효율성: 자원 관리  
안정성: 자원 보호  
확장성: 하드웨어 인터페이스 제공  
편리성: 유저 인터페이스 제공

# 1주차 2교시: 운영체제의 구조
## C3. 현대의 컴퓨터 환경
### (1) 커널과 인터페이스
**커널**: 프로세스 관리, 메모리 관리, 저장장치 관리와 같은 운영체제의 핵심적인 기능을 모아놓은 것.  
**인터페이스**: 커널에 사용자의 명령을 전달하고 실행 결과를 사용자에게 알려주는 역할 

#### ㄱ. 각종 사용자 인터페이스
**디스크**: DISK Controller  
**USB**: USB Controller  
**Monitor**: Graphics Adapter  

### (2) 시스템 호출과 디바이스 드라이버
**시스템 호출**: 커널이 자신을 보호하기 위해 만든 **인터페이스**  
- 커널은 사용자, 응용 프로그램으로부터 컴퓨터 자원을 보호하기 위해 자원에 직접 접근하는 것을 차단.  
- 사용자 모드와 커널 모드로 분리됨. 이 두 모드를 이어주는 것아 **시스템 호출 인터페이스**이다.  

#### ㄱ. 시스템 호출: 직접 접근
- 두 응용 프로그램이 자기 마음에 드는 위치에 데이터를 저장하려고 함.
- 다른 사람의 데이터를 지울 수도 있고 내 데이터가 다른 사람에 의해 지워질 수도 있음.

#### ㄴ. 시스템 호출: 시스템 호출을 통한 접근
- 응용 프로그램이 직접 하드 디스크에 데이터를 저장하지 않고, 커널이 제공하는 함수 `write()`를 통하여 데이터를 저장.
- 커널이 데이터를 가져오거나 저장하는 것을 전적으로 책임지기 때문에 컴퓨터 자원 관리가 수월함.

#### ㄷ. 시스템 호출 정리
- 시스템 호출은 커널이 제공하는 시스템 자원의 사용과 연관된 함수이다.
- 응용 프로그램이 하드웨어 자원에 접근하거나 운영체제가 제공하는 서비스를 이용하려 할때는 시스템 호출을 사용해야함.

#### ㄹ. 디바이스 드라이버: 
**디바이스 드라이버**: 커널과 하드웨어의 **인터페이스**를 담당하며 디바이스 드라이버라고 불림.
- 마우스 같은 간단한 제품은 드라이버를 커널이 가지고 있으나 그래픽 카드 같은 복잡한 하드웨어의 경우 제작자가 드라이버(S/W)를 제공함.


## D4. 운영체제의 구조
### (1) 커널의 구성
**커널**: 컴퓨터의 운영체제의 핵심이 되는 컴퓨터프로그램으로 시스템을 통제하는 역할을 함.

> Application &rarr; **Kernel** &rarr; CPU, Memory, Devices

#### ㄱ. 단일형 구조 커널
- 초창기의 운영체제 구조
- 커널의 핵심 기능을 구현하는 모듈들이 구분없이 하나로 구성

### (2) 계층형 구조 커널
- 비슷한 기능을 가진 모듈을 묶어 하나의 계층으로 만들고 계층 간의 통신을 통해 운영체제를 구현하는 방식
- `User Space` &lrarr; `Kernel Space (MicroKernel)` &rlarr; `Hardware`

### (3) 마이크로 구조 커널
- 프로세스 관리, 메모리 관리, 프로세스 간 통신 관리 등 가장 기본적인 기능만 제공

### (4) 가상머신
정의: 운영체제와 응용 프로그램 간에 작동하는 프로그램
- 가상머신을 설치하면 응용 프로그램이 모두 동일한 환경에서 작동하는 것처럼 보임
- 예시: Java는 유닉스와 윈도우에서 작동하는 다양한 가상머신을 제공하는데, 이를 JVM(Java Virtual Machine)이라고 함

**JVM**이 **MiddleWare**의 역할을 하여 `.java` 파일을 읽어들여 각 운영체제에 맞는 명령어로 전환함.

### (5) UNIX n Linux
**특징**: 이식성
리누스 토바즈가 UNIX 호환 커널을 만들어 무료로 배포함. `LINUX`

### (6) 모바일 운영체제
- **안드로이드**: 구글이 개발한 운영체제: Linux 커널을 사용하여 제작됨.: 무료로 배포됨  
- 
- **IoS**: 애플이 개발한 스마트폰 전용 운영체제  

이러한 모바일 시장의 발달로 마이크로소프트의 성장이 한풀 꺾임.
