# 2주차 1교시: 정보전송 시스템 개요
## L12. 정보전송 시스템 개요
정보전송 시스템: 데이터 전송계
- 단말장치
- 정보전송 회선
- 통신 제어장치

정보전송 시스템: 데이터 처리계
- 컴퓨터(중앙처리장치, 기억장치, 입출력 장치


### (2) 데이터 전송계
**단말장치**: 정보의 입력과 수신  
**신호변환장치**: 변조, 복조  
**통신회선**: 변환된 신호의 이동 통로나 통신망  
**통신제어장치**: 컴퓨터와 모뎀사이에 위치하여 송수신되는 데이터를 처리하기 좋은 형식으로 변환  

### (3) 데이터 처리계
**중앙처리장치**: 컴퓨터에서 입력되는 데이터를 가공, 처리, 축적, 수정, 변경 추가
**기억장치와 입출력 장치**: 컴퓨터에서 데이터를 저장하는 저장장치와 입출력 장치로 구성

### (4) 정보전송 시스템: 오프라인, 온라인, 즉시 처리, 배치 처리
- 오프라인 처리
  - 자료 등을 USB나 외장 하드 디스크에 저장하여 통신회선 없이 사람이 직접 전달하는 형태
- 온라인 처리
  - 자료 등을 통신회선을 통하여 한 장소에서 다른 장소로 전달하는 형태

- 즉시 처리(실시간 처리)
  - 자료를 처리하는 시점이 발생하는 즉시 처리하는 형태
- 자료를 처리하는 시점이 발생하는 즉시 처리하는 것이 아니라 모아두었다 처리하는 형태

## M13. 정보전송 시스템의 하드웨어
### (1) 단말장치 
#### ㄱ. 디지털 데이터를 입-출력 하는 데 사용하는 장치
- 키보드, 모니터, 프린터 등 컴퓨터와 연결되는 모든 주변 장치
- 단말기 또는 단말, 터미널

#### ㄴ. 단말장치의 구성과 기능
**입출력 제어부**: 외부에서 정보를 받아 외부로 출력함  
**송수신 제어부**: 데이터 송수신, 제어 문자 검색, 필요 동작 수행  
**오류 제어부**: 오류 검출 후 복구  
**공통 제어부**: 단말 장치를 종합적 제어  

#### ㄷ. 단말 장치의 종류와 기능
- 데이터를 입출력하는 단순한 기능에서 벗어나 점차 지능화
- 특수 용도에 맞게 발전
- 스마트폰이나 애플폰의 발전에 따라 정보 단말기도 소형화
- 휴대형 정보 단말기의 기술 특성인 고용량 전지, 대용량 메모리 IC, 저전력 RF 부품 기술 등이 발전

### (2) 신호 변환 장치
- 송신자의 정보를 전기적 신호로 변환한 후 전송매체(통신회선)를 거쳐 전송
- 전기적 신호는 수신자의 신호 변환장치를 이용해 원래의 정보로 변환
- DCE는 신호 변환 외 전송 신호의 동기 제어 송수신 확인, 전송 조작 절차의 제어 등을 담당

**송신자** &rarr;(정보)&rarr; **신호 변환 장치** &rarr;(전기신호)&rarr; **신호 변환 장치** &rarr;(정보)&rarr; **수신자**

#### ㄱ. 신호 변환 장치
**아날로그 신호**
- 비슷하다(Analogous)에서 파생
- 연속적으로 변하는 신호
- 전압이 여러개
- 자연스러움
- 음성, 음악, 텔레비전 영상, 그림, 사진 등등

**디지털 신호**
- 손가락(Digit)에서 유래
- 이산적인 신호
- 전해진 전압 값
- 인공적
- 단말장치, 컴퓨터, 팩스, 텔렉스

디지털 데이터를 MODEM을 통해서 아날로그 신호로 만들어 통신망으로 보냄.

### (3) 신호변환 장치의 종류
#### ㄱ. 모뎀
- Modulator, DEModulator의 합성어
- 송신부: 데이터 부호화기, 변조기, 스크램블러, 주팟수 체배기
- 수신부: 등화기, 대역 제한 여파기, 자동 이득 조절기, 복조기, 데이터 복호화기, 디스크램블러

#### ㄴ. 디지털 서비스 유닛 (DSU)
- 디지털망에 사용하는 회선 종단 장치(DCE)

#### ㄷ. 통신회선
- 데이터를 전송하는 통로
- 통신기기 사이를 연결하는 역할
- 데이터 전송선로 또는 정보전송 매체
- 데이터 전송선로는 유선과 무선으로 구분

꼬임선
- 구리선 두 가닥을 균일하게 꼬아 여러 다발로 묶어 보호용 피복선을 입힌 케이블
- 전자기 간섭을 줄이기 위해 꽈

동축 케이블
- 내부에 있는 단열 구리선과 외부 도체로 구성
- 내부 도체와 외부 도체 사이에는 절연물질이 있으며, 외부 도체는 피복으로 보호
- 유선 TV, 근거리 통신망, CATV 등 고주파 신호의 광대역 전송과 장거리 전화 전송에 주로 사용

광섬유 케이블
- 지름의 굵기가 0.1mm 정도인 석영(유리섬유)을 케이블 안에 여러가닥 넣어서 레이저 광의 전반사 현상을 이용해 데이터를 전송하는 원통형 선로
  - 꼬임선, 동축 케이블에 비해 대역폭이 넓어 고속으로 전송 가능

무선선로
- 지구의 대기 등에서 전자기파를 이용하여 데이터를 전송하는 비유도체
  - **마이크로파**: 방향성이 있음 (2~40GHZ); 강으로 분리된 지역에 유선선로 설치가 어려움
  - **라디오파**: 방향성이 없음 (30kHZ~1GHz); 방송용
  - **위성 마이크로파**: 지상에서 35,680km 상공에 위성을 띄워놓고 지상의 여러 송수신국을 서로 연결

정지궤도 위성
- ㅇㅇ.. 

이동통신
- 이동국, 기지국, 교환국으로 구성
  - 이동국: 모든 이동체에 설치된 통신 단말기
  - 기지국: 이동국과 기지국을 연결하는 기능
  - 교환국: 왕

통신 제어장치
- 주로 단말장치/컴퓨터와 모뎀 사이에 위치
  - NCU 네트워크 컨트롤 유닛
- 다중화: 여러 신호를 Multiplexer에 보냄
- 역다중화: Multiplexer에서 다중으로 보냄
- 전위처리기(FEP)
- 후처리장치(BEP)

통신 제어 처리 장치 (CCP)
- 프로그래밍을 사용하여 통신 제어장치를 개선
- 프로그램 가능한 또는 프로그램 제어 방식의 통신 제어장치
- 메시지 제어에 관한 부분까지 처리하기 때문에 CPU의 부담이 줄어듦

인터페이스
- RS-232, 가장 대표적인 인터페이스 표준화
- DTE, DCE 간의 물리적 연결과 신호 수준을 정의
- 25핀과 9핀 연결기 모두 지원

# 2주차 2교시: 통신 프로토콜의 개념과 설정 명령어
## N14. 통신 프로토콜 개념
**정의**: 컴퓨터나 원거리 통신 장비 사이에서 메시지를 주고 받는 양식과 규칙의 체계

- **구문 Syntax**: 전송하고자 하는 데이터의 형식(Format), 부호화(Coding), 신호 레벨(Signal Level)등을 규정
- **의미 Semantics**: 두 기기 간의 효율적이고 정확한 정보 전송을 위한 협조 사항과 오류 관리를 위한 제어 정보를 규정
- **시간 Timing**: 두 기기 간의 통신 속도, 메시지의 순서 제어 등을 규정



### (1) 물리적인 측면
물리 계층 &rarr; 데이터링크 계층 &rarr; 네트워크 계층
- 자료 전송에 쓰이는 전송매체, 접속용 단자 및 전송 신호, 회선 규격 등

### (2) 논리적 측면
전송 계층 &rarr; 세션 계층 &rarr; 표현 계층 &rarr; 응용 계층
- 프레임 구성, 프레임 안에 있는 각 항목의 뜻과 기능, 자료 전송의 절차 등

폐쇄적인 프로토콜
- 자사 장치들끼리 통신하기 위한 독자적인 통신 규약
- 자세한 규격이 공개되어 있지 않아서 크래킹 위협에 상대적으로 안전

공개된 범용 프로토콜
- 여러 장치들에 널리 알려진 규격
- 규격이 널리 공개되어 있기 때문에 컴퓨터와 네트워크 크래킹에 취약한 편

응용: HTTP, SMTP, FTP, Telnet  
표현: ASCII, MPEG, JPEG, MIDI  
세션: NetBIOS, SAP, SDP, NWLink  
전송: TCP, UDP, SPX  
넷웤: IP IPX  
데이터링크: Ethernet, Token Ring, FDDI  
물리: X  

### (3) 프로토콜 종류
#### ㄱ. FTP(20,21)
- 파일 전송 프로토콜
- 20: 데이터 전송용
- 21: 제어용
- 보안성 문제로 TLS와 결합안 FTPS와 SSH와 결합한 SFTP가 있음

#### ㄴ. TFTP(69)
- FTP보다 간단하지만, 전송 과정에서 손실될 수 있음

#### ㄷ. Telnet(23)
원격 컴퓨터 시스템에 로그인 할 수 있도록 도와주는 프로토콜

#### ㄹ. HTTP(80)
하이퍼 텍스트 전송 프로토콜

#### ㅁ. SNMP(161)
- 네트워크 관리 시스템 프로토콜
- 네트워크 관련 자원들을 효율적으로 감시하고 제어하는 기능 수행

#### ㅂ. SMTP(25)
- Simple Mail Transfer Protocol 

#### ㅅ. POP3(110)
- 메일을 저장하고 있다 사용자가 메일 서버에 접속하면 이를 보내주는 역할

#### ㅇ. DHCP(67,68)
- 클라이언트가 동적인 IP 주소를 할당받아 인터넷을 사요 ㅇ할 수 있게 해주는 프로토콜
- 67: BOOTP 서버
- 68: BOOTP 클라이언트

#### ㅈ. ICMP(1)
운영체제에서 오류메시지를 전송 받는데 주로 사용

#### ㅊ. NETBIOS(139)
- 컴퓨터 어플리케이션이 근거리 통신망을 통해 통신할 수 있게 해줌.

### (4) 프로토콜의 기능
#### ㄱ. 단편화와 재합성
- 단편화: Fragmentation, 송신 측에서 긴 데이터 블록을 손쉽게 전송할 수 있도록 크기가 작은 블록으로 나누어 전송
- 재합성: Assembly, 수신 측에서 쪼개진 작은 데이터 블록을 재합성하여 원래의 메시지로 복원하는 기능

#### ㄴ. 캡슐화
- 각 프로토콜에 적합한 데이터 블록을 만들기 위해 데이터 정보를 추가하는 것

#### ㄷ. 연결 제어
- 비연결 데이터 전송과 연결 위주 데이터 전송을 위한 통신로를 개설-유지-종결하는 기능

#### ㄹ. 흐름 제어
#### ㅁ. 오류 제어
#### ㅂ. 순서 결정
#### ㅅ. 주소 설정
#### ㅇ. 동기화
#### ㅈ. 다중화
#### ㅊ. 전송 서비스

## O15. 데이터 전송 규칙
최소한 컴퓨터가 두대는 있어야 네트워크라고 함
### (2) 패킷
**정의**: 컴퓨터 간에 데이터를 주고받을 때 네트워크를 통해 전송되는 데이터 전송단위
- 용량이 큰 데이터를 전송할 때는 작게 나누어서 보내는 것이 규칙
- 분할된 패킷을 수신지로 전송할 때 네트워크 상황에 따라 전송한 순서대로 도착하지 않을수도 있음

#### ㄱ. 패킷의 구조
패킷 - 헤더 - 페이로드 

## P16. 유무선 네트워크 설정 및 명령어
- hostname: 컴퓨터 이름 확인
- netuser: 컴퓨터 사용 계정 확인
- ping: Packet Internet Groper
    - 네트워크에 있는 호스트들의 상태를 점검
- ipconfig
  - 현재 로컬 컴퓨터의 설전된 네트워크 값 정보를 확인할 수 있는 명령어
- Tracert
  - 접속하려는 호스트까지의 경로 추적
