**클라이언트**: 서버에서 보내주는 데이터 서비스를 수신하는 컴퓨터

- 애니캐스트

#### ㄴ. 멀티캐스트

#### ㄷ. 브로드캐스트

## P16. 전송 방향에 따른 통신 방식
### (1) 개요
**통신**: 떨어져 있는 두 지점 간에 정보를 전송하는 것
- 두 장치 간에 데이터를 전송할 때는 데이터 전송 방향에 따라 단방향 통신과 양방향 통신으로 나눔
- 양방향 통신은 정보를 주고받는 시점에 따라 다시 반이중 통신과 전이중 통신으로 나눔

### (2) 단방향 통신 (Simplex)
- 통신 채널을 통해 접속된 단말기 두 대 사이에서 데이터가 한쪽 방향으로만 전송되는 통신 방식
- 단방향 통신에서 전기적으로 신호를 보내려면 송신 측과 수신 측을 연결하는 회로를 구성해야 함
  - 두개가 필요하겠죠?
- 라디오 아날로그 TV, 모니터, 키보드 등

### (3) 양방향 통신 (Duplex)
- 반이중 통신: 데이터의 송수신을 한번씩 번갈아 가면서 할 수 있음 (무전기)
- 전이중 통신: 데이터의 송수신을 동시에 가능 ()

#### ㄱ. 반이중 통신
- 통신 채널에 접속된 두 대의 단말기 중 어느 한쪽이 데이터를 송신하면 상대방은 수신만 할 수 있다요.
- 양쪽 단말기의 상호 협력에 따라 송수신 방향이 바뀜

## Q17. 직렬 전송과 병렬 전송
### (1) 개요
> 장치사이에 데이터를 전송할 때 고려사항
> - 한번에 전송? 나누면 몇번? 

### (2) 직렬전송
하나의 정보를 각 데이터 비트를 직렬로 나열한후, 하나의 회선을 사용하여 순차적으로 1비트씩 송신하는 방식

- 병렬전송에 비해 느림
- 물리적 거리가 길어지면 경제적

#### ㄱ. 동기식 전송
- 두 시스템간의 클럭차가 있기 때문에 동기화(Synchronization) 해줘야함.
- 송신 비트 간격(TS)와 수신 비트 간격(TR)의 차이

자세한 전송 방식
- 송신측은 **미리** 정해진 수만큼 묶어서 일시에 전송함.
- 수신측은 무자열을 수신후 디코딩

대표적인 예:
- 파일 업로드 및 다운로드

비트 지향 동기화 기법, 문자 지향 동기화 기법으로 나뉨

#### ㄴ. 비트 지향 동기화 기법
- 데이터의 시작과 끝을 알리는 Flag로 구성됨
- 01111100 와 0111110  

#### ㄷ. 문자 지향 동기화 방법
- 모든 데이터 단위를 문자 단위로 처리함
- STX와 ETX로 구성됨 (Start of teXT, End of teXT

#### ㄹ. 비동기식 전송
- 보통 시작 비트는 1비트를, 정지비트는 1~2비트 사용
- 문자는 5~8비트, 문자 비트열 뒤에 패리티 비트

단순 단거리 저속 통신에 적합하지만, 오버헤드가 많음.

### (3) 병렬전송
> 부호를 구성하는 비트 수와 같은 양의 통신회선을 사용하여 여러 데이터 비트를 동시에 병렬로 전송하는 방식

- 비트 n개를 전송하기 위해 회선 n개 사용
- 비싸서 짧게 구현함.

# 4주차 2교시: 통신 오류 검출
## R18. 통신 오류 검출
### (1) 개요
- 수신 측으로 전송한 데이터는 송신 측의 데이터와 동일해야 하지만, 다양한 원인으로 오류 발생함
- 종류에는, 단일 비트 오류, 다중 비트 오류, 집단 오류가 있음

#### ㄱ. 단일 비트 오류 (Single Bit Error)
- 데이터 단위 중 하나의 비트만 변경하는 오류

#### ㄴ. 다중 비트 오류 (Multiple Bit Error)
- 두개 이상의 **비연속적**인 비트가 변경되는 오류

#### ㄷ. 집단 오류(Burst Error)
- 두개 또는 그 이상의 **연속적** 비트가 변경되는 오류

집단 오류 검출 방식
- 패리티 비트 검사 Parity Bit Check
- 블록 합 검사 Blcok Sum Check
- 순환 중복 검사 Cyclic Redundancy Check  

## S19. 패리티 비트 검사]
### (1) 개요
- 전송하는 데이터 마다 패리티 비트를 하나씩 추가하여 홀수 또는 짝수 검사 방법으로 오류를 검출
  - 7비트 1비트 (1바이트) 패리티 검사
    - 패리티 비트를 홀수-짝수로 구분

### (2) 홀수 패리티 방식 (Odd)
- 전체 비트에서 1의 개수가 홀수여야함

### (3) 짝수 패리티 방식 (Even)
- 전체 비트에서 1의 개수가 짝여야함.
- 짝수 패리티는 패리티 비트를 포함하여 `XOR` 연산을 함.

### (4) 블록 합 검사
행단위 패리티에 열단위 오류 검사를 시행함
#### ㄱ. 개요
추가된 열 패리티를 ,`BCC`(Block Check Character)라고 함

## T20. 순환중복검사
> 정확하게 오류를 검출하기 위해 다항식 코드를 사용하는 방법

#### (1) 개요
- **다항식 polynomial**
  - CRC 발생기는 0과 1의 스트링보다는 대수 다항식으로 표현하며 하나의 다항식은 제수(DIVISRO)로 표현
