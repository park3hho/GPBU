# 5주차 1교시: OSI 7계층 프로토콜
## V22.통신 프로토콜의 개요
### (1) 통신 프로토콜
**정의**: 통신을 원하는 두 객체 간에 무엇을, 어떻게, 언제 통신할 것인지 서로 약속해 놓은 규정
- 정보통신에서 통신을 통제하는 규칙들을 규정

### (2) 통신 프로토콜의 발전과 표준화 배경
- 처음엔 IBM에서 제정한 프로토콜인 BSC와 SDLC를 많이 사용

X.25는 처음으로 패킷 단위로 교환하기 시작할 떄의 표준임

## W23. OSI 7계층 프로토콜과 참조 모델
> X.200으로 알려진 국제 네트워크 표준 규약
- 국제표준화 위원회에서 제정하여 6년의 개발 끝에 1983년에 완성

7계층 구성
- 물리
- 데이터링크
- 네트워크
- 전송
- 세션
- 표현
- 응용

## X24. 물리 계층
- 가장 최하위 계층
- DTE와 DCE의 실질적인 연결을 함
  - DTE: 데이터 단말장치, Data Terminal Equipment
  - DCE: 데이터 회선종단장치, Data Circuit Terminating Equipment

기능적 특성과 절차적 특정  
전기적 특성과 기계적 특성  

### (1) 물리 꼐층
> 물리적 연결에서 비트 단위의 데이터열을 전송하기 위해 신호와 부호화, 전송 방식 등을 구성

- 송신측은 0과 1의 비트열의 데이터를 전기 신호를 변환 후 전송 매체를 통해 수신 측으로 전달
- 수신측은 받은 전기 신호를 0과 1로 구성된 비트열로 복원하여 상위 계층에 전달

### (2) 대표적인 물리계층 프로토콜
ISO와 CCITT에서 결정 및 각 국가에서 결정
- V 시리즈, X 시리즈 

## Y25. 데이터링크 계층
> 2계층

대표 기능
- 노드-노드 전달, 주소 지정, 전송 제어, 흐름 제어, 오류 제어, 동기화
- 비트열 데이터로 프레임을 구성하여 하위계층으로 전달
- 데이터링크 계층은 데이터 동기를 제고하고 비트를 식별하는 기능을 제공

### (1) 노드-노드 전달 주소 지정
- 이웃 노드간의 데이터 링크를 설정하는 기능
- 전송할 데이터 앞에는 헤더, 뒤에는 트레일러를 추가하여 하위계층(물리계층)으로 전달
- 헤더와 트레일러에는 발신지 주소와 목적지 주소 정보가 담겨있음

### (2) 전송제어 기능
- 데이터 송수신 시 데이터를 올바르게 전송하는 일련의 절차를 제어함

교환회선의 전송제어절차
- 회선접속
  - 정보를 전송하기 전 단계
- 데이터링크 확립
  - 데이터의 송수신이 가능하도록 경로를 구성하는 단계
  - 폴링: 단말기로 컴퓨터 데이터를 전송하는데 사용
- 정보 전송
  - 링크 확립 후 데이터 전송
- 데이터링크 해제
- 회선절단

### (3) 흐름제어
양측의 통신 속도가 다를 때 조절하는 것.
- 정지 대기 방식: 하나씩만 전송 및 하나씩만 검사
- 슬라이딩 윈도 방식: ㅇㅇ 여러개 적절히 겹쳐서 보냄

### (4) 오류제어
- 오류 검출-수정-제어

반향 검사
- 루프검사

검출후 재전송 (ARQ)
- ACK NAK을 사용한 방식

FEC, 전진 오류 방식
- 수신 측에서 오류를 발견하면 발견 및 수정까지함, 재요청을 안해 역채널을 쓰지 않음
- 블록코드, 콘볼루션 코드

오류 검출 방식
- 패리티 비트 검사
- 블록 합 검사
- 순환 중복 검사
- 해밍 부호 검사

### 데이터링크 프로토콜 종류
- 문자지향방식
  - Kermit, BSC
- 비트 지향방식
  - 데이터를 비트 단위로 전송

# 5주차 2교시: OSI 7계층(2)
## Z26. 네트워크 계층
> 3계층

**전송 계층**에서 전달받은 데이터를 패킷으로 감싸 **데이터링크 계층**으로 전달

- 대표 프로토콜
  - CCITT의 X25가 있음
  - CCITT의 X.21, CCITT의 Q.931, 연결형 ISO 8473

- **경로 배정과 중계**: 통신망 내부에서 경로를 배정하고 중계함
- **상호 통신망 연결**: 다양한 통신망이 연결되어 있어도 통신망 계층의 기능을 함
- **통신망 접속**: 통신망의 연결을 설정하고 종료함
- **흐름 제어**: 네트워크 노드 사이에서트 래픽을 제어함
- **순서 제어**: 송신 측에서 데이터를 보낸 순서대로 수신측에서 받지 못할 때 순서를 제어함

## A1. 전송계층과 세션계층
### (1) 전송계층
> 4계층

- 송수신 측의 응용 프로세스 사이에서 **데이터를 확실히 송수신해 주는 역할**

**기능**
- 목적지의 주소 지정 (IP)
- 메시지 우선권
- 다중화
- 보안
- 계정
- 오류 발생 시 링크 회복
- 흐름 제어
- 송수신 할 때 무결성 보장

**역할**
- 등급 0: 최소한의 기능만 있는 간단한 프로토콜, 오류 통지
- 등급 1: 장애 시 기본 오류 회복
- 등급 2: 다중화 기능 부가
- 등급 3: 기능 1에 다중화 기능 추가
- 등급 4: 데이터 분실, 오류, 장애 시 오류를 검출하고 회복 및 다중화함.

### (2) 세션계층
> 5계층

송신 측과 수신 측에서 토큰을 사용하여 **연결-유지-해제**를 하는 역할
- 전이중과 반이중 방식으로 결정, 동기 관리 수행
- 대표 프로토콜
  - ISO 8327, CCITT의 X.225


## B2. 표현계층과 응용계층
### (1) 표현계층
> 6계층

- 송신 측과 수신 측 사이 서로 다른 부호체계 간의 변환과 표준화된 데이터 형식을 규정

### (2) 응용계층
> 7계층

- 사용자에게 직접 제공하는 서비스
- 전자우편, 파일 전송, 접근 및 관리, 데이터베이스, 네트워크 관리 도구 등등을 제공

## C3. 프로토콜 표준화와 표준화 기구
### (1) 구성요소의 모델화

**응용객체**: 연결을 도와주는 연결프로그램
**개방형 시스템**: 통신 기능을 제공
**물리 매체**: 물리적인 장치, 통신 회선 채널
**접속**: 응용객체 사이를 논리적으로 연결하는 장치

### (2) 계층
**개방형 시스템**: **표준**을 기본으로 서로 다른 시스템 사이에서도 통신이 가능하게 해주는 것
**부시스템**: 순서를 주는것

**프로토콜**: 동일 계층 간의 연결하는 것
**인터페이스**: 상위계층과 하위 계층을 연결하는것
**객체**: 시스템에서 각 계층의 일을 수행하는 모듈의 실체
**N-기능**: 객체가 수행하는 행동
**N-서비스**: 자신의 **상위 계층**에 제공하는 서비스
**N-서비스 접근점**: N 계층의 서비스를 제공받는 접점
**연결**: 진짜 연결

### (3) 프로토콜
- 분류 방식:
  - 단일 구조 및 계층형 구조
- **멀티포인트 혹은 간접 통신의 경우 통신 단위를 하나만 쓰기엔 복잡해서 계층 구조가 유리**

기능
- 단편화
- 재합성
- 캡슐화
- 순서 결정
- 연결 제어
- 흐름 제어
- 오류 제어
- 주소 설정
- 동기화
- 다중화
- 전송서비스

기본 표준: ISO 표준

표준 적용 위도
