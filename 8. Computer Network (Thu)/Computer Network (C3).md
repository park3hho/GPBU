# 3주차 1교시: 네트워크의 구성
## J10. 네크워크의 구성
### (1) 네트워크 구성 요소
- ㄱ. PC
- ㄴ. 접속장치
- ㄷ. 전송매체

#### ㄱ. PC(컴퓨터나 서버)
- 웹 브라우저를 통해 웹 서핑을 하거나 이메일을 주고받으면서 네트워크를 사용
- 카카오톡, 인스타그램은 모두 애플리케이션의 종류

#### ㄴ. 접속장치
- 네트워크 접속 장치는 애플리케이션의 데이터를 정상적으로 전송하기 위한 장치
- 네트워크 접속 장치에는 다양한 종류가 있지만 가장 기본적인 것으로 스위치와 라우터가 있음

스위치
- 한 **네트워크 내부**에서 데이터 전송을 수행하는 장치
- 컴퓨터를 스위치에 연결하는 것은 네트워크에 연결하는 것과 같은 의미

라우터
- 서로 다른 **네트워크를 구분** 짓고 연결하는 장치
- 대부분의 집은 네트워크가 하나뿐이지만 기업의 사내 네트워크에는 많은 네트워크가 존재함  
&rarr; 이러한 네트워크를 연걸하는 것이 바로 라우터임

#### ㄷ. 전송매체
- 컴퓨터와 네트워크 접속 장치는 전송매체에 의해 서로 연결
- 전송매체는 크게 유선전송매체와 무선 전송매체로 구분

유선 전송매체
- 일반적으로 케이블을 말함

무선 전송매체
- 전파를 말함
- 네트워크 통신에 이용되는 전파는 규격에 따라 전파의 파장이나 주파수가 지정되어 있음

## K11. 네트워크의 형태
### (1) 근거리 네트워크 (LAN, Local Area Network)
#### ㄱ. 근거리 네트워크
범위가 건물 안이나 특정 지역인 네트워크로 유선 케이블, 적외선 링크, 무선 송수신기 등을 이용하여 통신

- 근거리 네트워크는 일반적으로 전송 거리가 약 50m 정도의 거리임
- LAN은 작은 범위의 네트워크망으로 Ethernet, Token Ring, FDDI 형태로 구축함
- 지리적으로 제한된 곳에서 스캐너, 프린터, 컴퓨터 등을 연결할 수 있는 네트워크가 LAN임.

> 상황에 따라서 서로 다른 LAN을 함께 연결하는 것이 도움이 됨.

예) 같은 학과에 속해 있는 연구실 2개는 각각의 LAN을 사용할 때, 연구실 간의 데이터 통신이 필요할 때 사용

**LAN의 주요목적**  
자원 공유
- 원격지의 자원을 공유
- 복수의 사용자에게 독점적 사용권을 부여
- 사용자는 불편 없이 모든 자원을 사용 가능

분산 처리
- 독립된 각 장비에서 계산과 작업을 처리
- 전체 시스템의 능력은 연결된 모든 컴퓨터의 능력에 따라 결정

분산 제어
- 분산된 지역의 독립적인 장치 간의 통신을 통하여 프로세스를 제어
- 높은 데이터 전송 속도와 신뢰도 유지

정보 교환
- Video 및 Voice 신호 전달
- Text 데이터를 전달

**LAN 통신의 단점**  
- 전송 거리가 짧아서 거리에 제약이 있음
- UTP 및 광케이블로 구축되지만, 네트워크에 노드가 많아지면 충돌이 발생하여 성능이 떨어짐

### (2) 광역 네트워크 (WAN, Wide Area Network)
두 개 이상의 **근거리 네트워크**를 넓은 지역에 걸쳐 연결하는 것

### (3) 인트라넷 (Intranet)
인터넷에서 사용하는 회선과 여러 기반 기술을 이용하여 구축하는 사설 네트워크

- 분산된 캠퍼스간 ISP(인터넷 서비스 제공업체)까지만 연결하면 되기 때문에 저렴한 비용을 사설 네트워크를 구축할 수 있음.

## L12. 네트워크 접속 장치
### (1) 네트워크 접속장치 개요
네트워크 구성에서 가장 기본이 되는 하드웨어

- 물리적으로 떨어져 있는 다양한 형태의 근거리 네트워크를 연결하면 좀 더 복잫바고 큰 광역 네트워크를 형성할 수 있음
- 컴퓨터나 네트워크 장치는 NIC를 사용하여 네트워크에 연결할 수 있음
- 허브, 중계기, 스위치, 게이트웨이, 라우터 등을 사용하여 네트워킹 문제를 해결

### (2) LAN 카드 
- 두 대 이상의 컴퓨터로 네트워크를 구성하려고 외부 네트워크와 빠른 속도로 데이터를 송수신할 수 있게 컴퓨터 내에 설치하는 확장 카드

MAC 주소
- Media Access Control Address
- LAN 카드에 할당된 48비트의 물리적 주소

LAN 카드의 MAC 주소
- [시작]-[모든 프로그램]-[보조 프로그램]-[명령 프롬프트(cmd)]
  - cmd에서 입력 `ipconfig all`

### (3) 허브
여러 개의 입력 및 출력 포트가 있는 특수한 형태의 네트워크 장치

- 네트워크에 연결할 컴퓨터가 두 대 뿐일 때는 허브를 사용하지 않고 두 컴퓨터의 네트워크 카드를 케이블로 직접 연결하면 됨
- 연결할 컴퓨터가 세 대 이상이라면 허브를 사용해야 편리하게 연결할 수 있음.

#### ㄱ. 더미허브
그냥 일반적인 허브 많이 연결하면 전송속도가 떨어짐.

#### ㄴ. 스위칭 허브
- 전체 대역폭을 각 컴퓨터 노드 수만큼 나눠서 사용하는 문제점을 해결한 장치
- 단순히 전송하는 기능을 넘어 수신지의 주소로 스위칭하는 기능이 있음

#### ㄷ. 스태커블 허브
- 스택 접속 포트가 갖춰진 허브로, 허브와 허브 사이를 연결하여 용량을 확장할 수 있음
- 중규모 이상의 네트워크를 구성할 때 이 허브를 여러 층으로 쌓아서 구성
- 여러 대의 더미 허브를 연결하면 컴퓨터 간의 데이터 전송 속도 차이가 발생하지만, 스태커블은 논리적으로 하나처럼 동작함

#### ㄹ. 인텔리전트 허브
- 신호의 조절과 변경 등 다양한 지능형 기능을 포함한 허브

### (4) 스위치
컴퓨터에 할당되는 대역폭을 극대화하는 장치
- 허브와 달리 근거리 네트워크가 제공하는 대역폭을 컴퓨터로 모두 전송

### (5) 브리지
두 개 이상의 근거리 네트워크를 연결하여 하나의 네트워크로 만들어주는 장치
- 근거리 네트워크에서 하나의 장치가 데이터를 송신할 때 다른 장치도 데이터를 송신하면 충돌이 발생함
  - 네트워크에서 노드 수가 많아질 수록 충돌이 발생할 확률이 높고 통신 속도도 떨어짐
    - 브리지를 이용하면 이러한 문제를 해결할 수 있음
- 스위치와 비슷한 역할을 하지만, 전체 네트워크의 트래픽을 줄이기 위해 네트워크를 세그먼트 단위로 분할한다는 점이 스위치와 다름

### (6) 게이트웨이
- 서로 다른 두개 이상의 네트워크를 상호 접속하여 정보를 주고받을 수 있는 장치
- 접속할 수 있는 네트워크
  - LAN, PDN, PSDN(공중전화 교환망)
- 브리지와 달리 서로 다른 네트워크 간에도 프로토콜을 변환하여 정보를 주고받을 수 있음

### (7) 중계기
접속 시스템의 수를 증가시키거나 네트워크 전송 거리를 연장하려고 사용하는 장치 (리피터)

### (8) 라우터
서로 다른 네트워크 간에 통신하는 데 사용하는 장치
- 라우팅 테이블을 통해서 라우팅함.

특징
- 오류 패킷의 폐기 기능과 혼잡 제어 기능을 수행
- 논리 주소에 따라 네트워크 트래픽의 방향을 지정
- 둘 이상의 네트워크를 연결하는 데 사용
- 웹 페이지 요청을 수신지로 전송하는 데 결정적인 역할을 수행

# 3주차 2교시: 네트워크 접속 형태와 전송매체
## M13. 네트워크 접속 형태
### (1) 네트워크 전송 형태 개요
전송매체와 프로토콜이 결정되면 다양한 형태로 컴퓨터를 구성할 수 있음
- 네트워크의 구성을 '네트워크 토폴로지'라고도 함
  - 네트워크에 연결되어 있는 노드와 링크가 물리적 또는 논리적으로 배치되어 있는 방식을 말함

- 노드는 네트워크에 연결된 주소가 있는 통신장치를 말함
  - 컴퓨터, 프린터, 복합기 등이 하나의 노드가 될 수 있음
- 링크 하나에 2개 이상의 노드가 연결되며, 2개 이상의 링크가 접속 형태를 구성

성형, 트리형, 버스형, 링형, 그물형, 싱하형

### (2) 성형
가장 일반적인 네트워크 구성 형태
- 허브가 네트워크 중앙에 위치하여 다른 모든 노드를 연결
- 모든 노드가 중앙의 허브에 연결되어 통신하므로 통신망의 처리능력과 신뢰성은 허브가 좌우

**장점**
- 설치 비용이 저렴, I/O를 한놈만 책임지니
- 유지보수와 확장이 용이
- 결함을 쉽게 식별하고 분리하기 좋음

**단점**
- 중앙에 있는 제어장치의 가용성이 중요함
- 비교적 케이블이 많은 편

### (3) 버스형
모든 네트워크 노드와 주변장치가 파이프 등의 일자형 케이블에 연결되어 있는 형태
- 하나의 긴 케이블이 네트워크의 모든 장치를 연결하는 중추 네트워크 역할을 수행
- 모든 노드는 하나의 케이블에 연결되어 있고, 케이블의 시작과 끝에는 터미네이터라는 장치를 붙여서 신호가 케이블로 되돌아오는 것을 막음

**장점**
- 설치가 간단하고 케이블 비용이 적게 듦
- 장비를 추가하기 쉬움
- 고장이 나도 전체 네트워크 영향을 미치지 않음

**단점**
- 노드가 많아지면 성능저하 급격함.
- 케이블 가용성 중요
- 재구성이나 결합, 분리가 어려움
- 병행성에 취약함

### (4) 트리형
성형의 변형이라고도 함.
- 계층적인 네트워크에 적합, 상위 계층의 노드가 하위 노드들을 직접 제어하는 계층적인 네트워크

**장점**
- 제어가 간단함
- 중앙에 있는 하나의 전송제어장치에 더 많은 장비를 연결할 수 있어 각 장비 간의 데이터 전송 거리를 늘릴 수 있음
- 여러 컴퓨터로 분리하거나 우선순위 정할 수 있음

**단점**
- 중앙에 트래픽이 집중되어 병목현상이 발생할 수 있음
- 중앙의 전송제어장치가 다운되면 전체 네트워크에 장애가 발생

### (5) 링형
노드가 링에 순차적으로 연결된 형태, 모든 컴퓨터를 하나의 링으로 연결
- 각 노드들은 인접한 노드 두 개 하고만 연결됨,
- 단순 링형: 단방향 통신
- 이중 링형: 양방향 통신

**장점**
- 구조가 단순하여 설치와 재구성이 용이함
- 장애 발생 시, 복구 시간이 빠름
- 케이블 비용 절약 가능

**단점**
- 원거리 통신시 비용 발생이 큼
- 링 제어하는 절차가 복잡
- 가용성이 중요, 하나의 링 결함에 전체  네트워크 사용 불가능

### (6) 그물형
중앙에 제어하는 노드 없이 모든 노드가 상호 간에 전용의 점대점 형태로 연결되어 있는 형태
- 그물형에서는 n(n-1)/2의 물리적 채널 필요
- 네트워크가 복잡하고 많은 통신 회선이 필요
- 신뢰성이 높음

**장점**
- 통신이 원활함
- 가용성, 신뢰성이 높음
- 보안 유지가 좋음
- 결함 식별과 분리가 쉬움

**단점**
- 개비쌈
- 설치와 재구성이 빡셈

## N14. 유선 네트워크 전송 매체
### (1) 동축 케이블, Coaxial Cable
- 꼬임선보다 주파수가 높고 더 빠름
- 두개의 전도체로 구성
  - 하나는 케이블 가운데 있는 와이어
  - 다른 하나는 와이어를 감싸는 차폐용 그물망
  - 절연체는 이들 중간에 들어 있음

### (2) 꼬임선, 이중 나선 케이블, Twisted Pair
꼬임선은 플라스틱으로 덮인 두 가닥의 절연된 구리선을 나선형으로 꼬아서 만듦
- 한 쌍이 하나의 통신회선 역할을 하며, 여러 개의 쌍이 묶여 하나의 케이블을 형성하고 보호용 외피로 이를 감싸 완성
- 구리선을 꼬는 이유
  - 두 선 사이 전기적 간섭을 최소화하려고
- 성형 구성에서 가장 많이 사용함
- 싼 편
- UTP, FTP, STP 등등에 사용

#### ㄱ. UTP
- 가장 많이 사용하는 네트워크 케이블, Cat 100Mbps 속도로 전송 가능
#### ㄴ. FTP
- 전자기 간섭을 줄이려고 전체 케이블에 피복을 씌운 형태
- STP는 은박지까지(알루미늄)

### (3) 광섬유 케이블, Optical Fiber Cable
- 머리카락보다 가늘어 휘어지는 전송매체

## O15. 무선 네트워크 전송 매체
### (1) 라디오파, Radio Wave
- 빛의 속도로 데이터 전송 가능
- 진공상태에서도 가능
- 방향성이 없음

AM, 진폭 변조
- 신호의 강약을 조절하여 데이터를 보내는 방법

FM, 주파수 변조
- 주파수를 변조하여 데이터를 보내는 방법

### (2) 마이크로파 Microwave
극초단파, 센티미터파, 밀리미터파

- 주파수가 매우 높은 전파
- 통신과 레이더 등에 사용
- 직진, 흡수, 반사의 성질이 있음 
- 광파와 성질이 유사하당께
