# 1주차 1교시: 알고리즘 개요

## A. 알고리즘 개요

### (1) 알고리즘 개요

**Algorithm**: 9세기경 페르시아 수학자 al-Khawarixmi에서 유래됨.

**가장 오래된 알고리즘**: 기원전 300년경에 만들어진 최대공약수 Euclid의 알고리즘  
큰 수에서 작은 수를 뺀 수와 작은 수의 최대공약수와 같다.

### (2) 컴퓨터 알고리즘  

**1936 앨런 튜링의 논문** (On Computable Numbers, with an Application to the Entscheidungsproblem)  
모든 산술적 논리에 참과 거짓을 결정할 수 있는 방법이 존재한가? => 결정문제

**튜링 기계(컴퓨터)의 구성**  
`Tape`: 일정한 크기로 나뉘어 있는 무한한 테이프  
`Head`: 테이프에 제한된 숫자 기호를 읽거나 쓰거나 지울 수 있는 장치  
`State Register`: 장치의 내부 상태를 표현하는 기계  
`Action table`: 기호와 일의 순서를 정의하는 **규칙표** (알고리즘의 기초)   

### 3) 컴퓨터 알고리즘의 요건

**입력**: 0개 이상의 외부 입력  
**출력**: 1개 이상의 출력   
**명확성**: 각 명령은 모호하지 않고 명확해야 함.  
**유한성**: 일정 시간에 반드시 종료되어야 함.   
**유효성**: 모든 명령은 컴퓨터에서 수행 가능해야함. 

## 알고리즘의 생성 단계 

### B. 알고리즘 생성단계
#### (1) 생성 단계
설계 &rarr; 표현 &rarr; 정확성 검증 &rarr; 효율성 분석

(1) **설계**: 상향식 설계, 하향식 설계

(2) **표현**: 알고리즘을 표현하는 방법
- 자연어, 순서도 의사코드, 프로그래밍 언어

(3) **정확성 검증**
- 알고리즘이 타당한 입력이 주어졌을 때 유한시간대에 계산되어 올바른 결과를 출력하는 것

(4) **효율성 분석**
- 저장 공간과 연산수행의 효율성

### C. 탐색 알고리즘

#### (1) 순차 탐색 
정의: **순서대로** 탐색하는 것.  
시간 복잡도: O(N)

#### (2) 이진 탐색
조건: 정렬되어 있어야함.
정의: Binary Search, 중간 값을 탐색  
시간 복잡도: O(logN) + 1

#### (3) 탐욕 탐색 &rarr; 탐욕 알고리즘
정의: 매 선택에서 가장 최적의 답.  
시간 복잡도:
   - 정렬: O(n logN)  
   - 탐색 및 선택: O(n logN)

예시: 거스름돈을 가장 적은 동전 개수로 받을 수 있도록 하는 것.
> 500원 먼저, 100원, 50원, 10원 ...

# 1주차 2교시: 알고리즘의 표현 방법

## A. 소프트웨어 개발

소프트웨어 개발: 소프트웨어 기능 정의(요구사항 분석), 설계, 프로그래밍, 테스트, 유지보수  
프로그램 개발: 기능을 만들기 위한 개발, **설계(알고리즘)**, 코딩  
코딩: 컴퓨터가 실행할 수 있도록 절차와 규칙에 맞게 코드를 작성 

## B. 표현 
설계한 알고리즘을 표현하는 방법
- 자연어
- 순서도
- 의사코드
- 코딩(프로그래밍 언어)

### (1) 자연어
**정의**: 인간의 언어로 표헌, 가장 표현하기 쉬운 방법

### (2) 순서도
**정의**: 컴퓨터로 처리해야 할 작업 과정을 약속된 기호를 사용하여 순서대로 그림으로 표현

특징:
- 알고리즘의 이해와 추적이 쉬움
- 에러 발생시 시, 논리적 파악으로 위치 파악이 쉬움.
- 심벌의 형태, 이름, 의미를 이용하여 표현하며, 국제 표준이 있음.

| 구분       | 이름                   | 소재공학 / 재료공학                             |
|----------|----------------------|-----------------------------------------|
| 둥근 직사각형  | 시작/끝                 | 시작과 끝                                   |
| 각진 직사각형  | 처리 (Process)         | 지정된 동작, 연산, 값 데이터의 이동 등 모든 처리과정         |
| 마름모      | 판단 (Decision)        | 조건 비교후 참/거짓값에 따라 이동                     |
| 평행사변형    | 입력/출력 (Input/Output) | 일반적인 입출력                                |
| 원        | 연결자 (Connector)      | 흐름이 다른 곳으로 연결되거나 다른 곳으로부터 연결되는 입구를 나타냄. |
| 잘려진 직사각형 | 프린트 (Printing)       | 프린터를 통해 출력함                             |
| 화살표      | 흐름선 (FlowLine)       | 명령의 흐름을 나타내며, 화살표 방향의 순서대로 진행됨을 나타냄.    |

**순서도의 3가지 표현 방법.**   
(1) 순차형; (2) 선택형; (3) 반복형;


### (3) 의사코드
**정의**: 수도코드(Pseudo Code)라고도 함.
- 컴퓨터 공학에서 알고리즘이나 자료구조를 표현할 때 사용
- 정형화된 문법적 측면을 배제하고 사고의 흐름을 간결하고 효과적으로 전달하는 방법
- 특정 프로그래밍 언어와 관계 없이 논리의 명시를 위해 작성
- 일반적인 언어로 `코드를 흉내내어` 알고리즘을 표현

### (4) 코딩(프로그래밍 언어)
- 컴퓨터가 수행할 수 있는 명령어들의 집합
