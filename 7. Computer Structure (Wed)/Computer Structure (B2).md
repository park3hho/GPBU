# 1주차 2교시: 데이터 표현의 진법과 진법 변환

## E5. 디지털 정보 단위
### (1) 비트, Binary Bit
- 정보의 최소 단위
- 2진수

### (2) 니블, Nibble
- 4비트로 나타내는 정보 단위

### (3) 바이트, Byte
- 8비트로 나타내는 정보 단위
- 주소 지정의 단위로 사용됨.
- 용량의 기본 단위

### (4) 워드
- 컴퓨터가 한번에 처리할 수 있는 명령 단위
- Word: 2Byte (16bit CPU)
- Double-Word: 4Byte(32bit), 4Byte(64bit)
- Quad-Word: 4Byte(32bit), 8Byte(64bit)

MSB: 가장 높은 비트  
LSB: 가장 낮은 비트  

### (5) SI단위와 IEC 단위 비교
SI(10진수 단위) &rlarr; IEC(2진수 단위)

- $2^10=1,024$ / **Ki, K** / $10^3=1,000$  
- $2^20=1,048,576$ / **Mi, M** / $10^6=1,000,000$
- $2^30$ / **Gi, G** / $10^9$
...


## F6. 진법
### (1) 10진법
10진법 개념: 10진수 시스템에서 0~9까지의 10개의 숫자로 표현
- 기수는: 10
- 가중치(weight) 구조  
예: 480을 가중치로 표현 &rarr; (4x$10^2$)+(8x$10^1$)

### (2) 2진법
2진법 개념: 0과 1로 표현
- 기수는 2

### (3) 8진법
IPv4 표현에서 응용됨.
- 기수는 8

### (4) 16진법
- 0~9, A~F까지의 숫자, 문자로 표현함
- 컴퓨터 구조, 어셈블리어 &rarr; 자릿수를 더 짧게 표현 가능함.

### (5) 진법의 변환
#### ㄱ. x진법 &rarr; 10진법
소수점 이상은 나누기  
소수점 이하는 곱하기  

#### ㄴ. 2진수의 상호변환
2진수의 세자리는 8진수의 한자리  
2진수의 네자리는 16진수의 한자리  

## G7. 10진 정수 표현
### (1) 언팩 10진 형식
**정의**: 1바이트를 존(zone)부분과 디짓(digit) 부분 구성
- 존 부분: 항상 F(1111)추가
- 디질?: 10진수 값이 8421 BCD 코드 형식 표시

**부호표시**: 가장 오른쪽(하위)바이트의 존 부분에 부호 표시
- 양수(+):1100(C); 음수: 1101(D)
- 부호가 없을 때: 1111(F)

#### ㄱ. 예시
**+2018**  
16진수 표현: F 2 F 0 F 1 C 8

**-2018**  
16진수 표현: F 2 F 0 F 1 D 8

**2018**  
16진수 표현: F 2 F 0 F 1 F 8

### (2) 팩 10진 형식
#### ㄱ. 언팩 10진 형식, 연산 전에 팩 10진 형식으로 변환시켜 연산
- 연산 결과 출력할 때 다시 언팩 10지수로 변환
- 1 바이트에 2개의 디짓을 각각 8421 BCD 코드 표시

#### ㄴ. 부호표시: 각 오른쪽(하위)바이트 하위 4비트 부호 표시
- ASCII: 양수는 1010(A), 음수는 1011(B)
- EBCDIC(아스키코드 확장판)
  - 양수는 1100(C), 음수는 1101(D)

**+2018**  
16진수 표현: 0 0 0 2 0 1 8 A

**-2018**  
16진수 표현: 0 0 0 2 0 1 8 B

**2018**  
16진수 표현: 0 0 0 2 0 1 8 F


## H8. 실수 표현
### (1) 컴퓨터 과학적 표기방식, 2진 부동 소수점 표현
- 표준: IEEE 754 표준  
- **실수의 부동 소수점 표현**
  - $R = +-(0.m1,m2,m3,...mn)*be$
  - 부호(sign), 가수/소수부(mantissa), 기저/기수, 지수
- **표현 내 필수 요소**: 부호, 가수, 지수
- 단정도(single precision)/배정도(double precision)
  - 정밀도 = 유효자리수

### (2) 정규화: 과학적 표기방법
75.6875 = 1001011.1011(2)
    = 1.0010111011(2) * 2^6
    = 1.0010111011(2) * 2^110(2)

bias: 지수의 양수, 음수를 나타내기 위한 방법
- 표현 지수 = 바이어스 + 2진 지수 값
- 127(단정도), 1023(배정도)

### (3) 단정도(single Precision) 표현법
Bias(127) = 부호(1bit) + 지수(8bit) + 가수(23bit) 
#### ㄱ. 수치표현
- 가수 부분 정규화
  - 정규화 하면 항상 가수가 1임

### (4) 배정도(dobule Preicision) 표현법
Bias(1023) =  부호(1bit) + 지수(11bit) + 가수(52bit)
- 16자리까지 가능
