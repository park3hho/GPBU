# 1주차 1교시: 컴퓨터(시스템)의 일반구조 구성

## A. 컴퓨터(시스템)의 구성

### (1) 하드웨어:
- CPU, 
- Memory(주기억장치), DISK(보조기억장치) 
- IO(입출력장치)

### (2) 소프트웨어:
- 시스템 소프트웨어
  - 운영체제, 언어 번역 
  - 프로그램(컴파일러, 인터프리터 등등)
  - 유틸리티 프로그램(드라이버, 백신 등등)
- 응용 소프트웨어

### (3) 하드웨어
시스템 버스를 통해 각 구성요소들을 연결한다. 

#### ㄱ. **중앙 처리 장치** (Central Processing Unit)
- 컴퓨터의 특성을 결정
- 마이크로프로세서가 들어간 보드를 "**마이크로 컨트롤러**"라고 함.

**구성**: ALU, CU(Control Unit), Instruction Code, Register, Data Bus ....  
**Register**: 연산을 위해 잠시 데이터를 보관하는 `임시기억 장치`

**데이터 부분**: 연상장치, 레지스터  
**제어 부분**: 제어장치

#### ㄴ. 주기억 장치(main memory)
**ROM**: 비휘발성 메모리  
**RAM**: 휘발성 메모리

#### ㄷ. 보조기억 장치(Auxiliary Storage Unit)
하드 디스크, SSD, CD-ROM ...

#### ㄹ. 메모리 계층 구조
1. 레지스터 | Register
2. 캐시 | Cache
3. 메인 메모리 | Memory
4. 보조기억 장치 | Auxiliary Storage Unit

#### ㅁ. 시스템 버스
종류: 주소 버스, 데이터 버스, 제어 버스

- **데이터 버스**:
  - 프로세서, 메인 메모리, 주변 장치간 데이터 전송
  - 전송할 수 있는 비트 수를 결정
  - 메모리 버퍼 레지스터(MBR)연결, 양방향

- **제어 버스**:
  - 프로세서가 시스템의 구성요소를 제어하는데 사용하는데 사용
  - 제어 신호가 연산장치의 연산 종류, 메모리 R/W 동작결정
  - CPU(제어장치),메모리, 주변장치 간 양방향

- **주소 버스**:
    - 메모리에 데이터를 R/W 할 때 위치정보(주소) 전송
    - 배선수는 프로세서와 연결하는 메모리의 최대용량 결정
    - 메모리 주소 레지스터(MAR)와 연결, 단방향

- **인터페이스 회로 및 제어기**
    - `주소 버스`와 `데이터 버스`를 제외한 것은 모두 `제어 버스`이다.

- **버스의 대역폭 (bandwidth)**
  - 대역폭: 데이터버스를 통해 단위 시간당 전송 데이터크기
  - 단위: byte/sec
  - 버스대역폭 = 버스의 클록주파수 * 데이터버스의 폭
    - 버스의 클록 주파수: 100Mhz &rarr; 10nsec
    - 데이터버스의 폭 32Bit(4byte)
    - 4byte/10x10e-9sec = 400Mbyte/sec
  - 클록: Tick을 이용한 일정한 시간간격. 작업주기.
    - Tick: 
      - Pules
      - Clock Tick
  - Hz
    - 정의: 클록틱이 발생하는 속도를 나타내는 단위
- **FSB(FrontSideBus), BSB(BackSideBus)**
    - 전면 버스: 시스템버스, 메모리와 주변장치를 연결
      - 메모리는 FSB의 속도로 작동
  - 후면 버스: CPU 내부 버스, CPU 내부 장치 연결
    - CPU는 BSB의 속도로 작동
  - 두 속도의 차이는 Cache를 통해 해결함.  
> CPU &lrarr; ASIC &lrarr; Memory  

FSB: North Bridge, ASIC과 Memory 및 ASIC과 CPU를 연결함.  

#### ㅂ. 컴퓨터 시스템의 수행기능
- 프로그램 실행
- 데이터 저장
- 데이터 이동
- 데이터 입출력
- 제어

### (4) 소프트웨어
시스템 소프트웨어  
응용 소프트웨어  

#### ㄱ. 시스템 소프트웨어: OS
- 하드웨어를 관리, 응용 소프트웨어를 실행하는 역할
- OS

#### ㄴ. 시스템 소프트웨어: 장치 드라이버
- 컴퓨터 주변장치 구동/제어에 필요한 프로그램

#### ㄷ. 시스템 소프트웨어: 언어 번역프로그램
고급언어 프로그램 
- 인터프리터, 컴파일러 

#### ㄹ. 시스템 소프트웨어: 링커
- 프로그램에 생성된 목적 프로그램 + 라이브러리 루틴
  - 실행 가능한 하나의 프로그램 연결
- 연결 편집기(Linkage Editor)
  - 아, 이거 그 라이브러리 불러와서 할 수 있게끔 해주는 놈.

#### ㅁ. 로더
- 저장장치에서 보관된 프로그램을 주기억장치에 적재하는 역할

### (5) 프로그램 처리 과정
고급 언어 &rarr; 어셈블리어 &rarr; 기계어

#### 연산코드와 오퍼랜드 (opcode and operand)
**연산코드**(opcode): CPU가 수행할 연산을 지정하는 비트들
  - 예시: 비트 수 = 3; 지정할 수 있는 연산의 최대 수: **$2^3$**     

**오퍼랜드**(operand): 연산에 시용될 데이터가 저장되어 있는 기억 장치 주소  
  - 예시: 비트 수 = 5; 주소 지정할 수 있는 기억 장소의 최대 수: $2^5$ 

#### 프로그램과 데이터가 주기억 장치에 저장되어 있는 형태
- 명령어와 제이터는 지정된 기억 장소에 저장
- **워드(word)** 단위로 저장됨.
  - 워드는 CPU에 의해 한 번에 처리될 수 있는 비트 그룹
  - 예시: 8, 16 , 32, 64

## A1. 컴퓨터의 발전 과정
### (1) 초기의 계산 도구
- 주판: `5진법` 최초로 사용된 계산 도구 (메소포타미아인들이 사용함.)

### (2) 기계식 계산기
- 파스칼 계산기 (1642년): 덧셈/뺄셈 가능, 톱니바퀴를 이용한 수동 계산기
- 라이프니치 계산기 (1672년): 곱셈/나눗셈 가능, `2진법`
- 차분 기관(1823년), 해석 기관(1833년): 영국 배비지 개발, 삼각함수, 유효숫자 5자리까지 계산, 방정식을 순차연산 가능
- 천공 카드 시스템(1899년): 미국의 홀러리스, 인구조사 활용

### (3) 전기 기계식 계산기
- MARK-I(1944년): 하버드 대학의 에이킨 개발, 미해군 탄도 계산

### (4) 전자식 계산기
- 에니악(1946년): ENIAC, 에커트와 모클리, 진공관(트랜지스터) 사용, 최초의 전자식 컴퓨터
- 에니삭(1949년): EDSAC, 윌키스, 10진수 프로그램 내장 방식 계산기
- 에드박(1951년): EDVAC, 폰노이만, 2진법, 프로그램 내장 방식 적용
- 유니박(1951년): UNIVAC, 모클리와 에커트, 최초의 상용 컴퓨터

## B2. 컴퓨터의 세대별 분류
### (1) 1세대 컴퓨터
특징: 
- 진공관 사용, 컴퓨터 크기 &uarr;, 발열 &uarr;, 전력 소모 &uarr;
- 폰 노이만 제안, 프로그램 내장의 개념을 도입
- 수치 계산, 통계 등에 사용
- 대표: ENIAC, UNIVAC, EDSAC, EDVAC
- 언어: 기계어, 어셈블리어

### (2) 2세대 컴퓨터
특징:
- 대용량 보조 기억 장치 사용
- 운영체제 개념의 도입
- 다중 프로그래밍 기법/ 온라인 실시간 처리 방식 도입
- 과학 계산/ 일반 사무용으로 사용
- TR 사용, 컴퓨터 고속화, 기억 용량 &uarr;, 소형화
- SW 개발 주력한 시기 FORTRAN, ALGOL, COBOL 등

### (3) 3세대 컴퓨터
특징:
- 주요소자: SSI, MSI (IC 소자의 사이즈)
- 언어: LSIP, PASCAL, BASIC ...
- **IC개발**: 캐시(cache) 기억 장치가 등장
- OMR, OCR, MICR 도입
- 시분할 처리: 멀티프로그래밍 지원
- 다중모드 시스템 등장
  - 일괄처리, 시분할 처리, 실시간 처리, 다중 프로그래밍 제공
- TCP/IP 통신 표준화 (네트워크 관련 개념 도입)
- 경영 정보 시스템이 확립

### 4세대 컴퓨터
특징:
- 주요소자: LSI, VLSI
- 언어: ADA ...
- 고밀도 집적회로(LSI)
- 초고밀도 집적회로(VLSI)
- 마이크로프로세서 개발
- 멀티미디어 실용화
- 인공지능의 개발
- 사무자동화(OA), 공장자동화(FA), 가정자동화 완료(HA), 개인용 컴퓨터의 등장

### 5세대 컴퓨터
특징:
- 주요소자: VLSI, ULSI
- 언어: C++, Java ...
- VLSI, ULS 소자: 초소형, 초고속 추구
- 경영 정보, 지식 정보, 인공지능, 신경망, 퍼지, 멀티미디어, 가상현실을 목표
- GUI 환경 확장
- 다중 프로세서: 병렬 처리/광 컴퓨터/신경망 컴퓨터 개발
- 인공지능의 연구 진행
- **비 폰 노이만 Non-Von Neumann 컴퓨터 구조 제안**
- 객체 지향 프로그래밍 언어 사용

### IC, Integrated Circuit(집적 회로)의 분류
**SSI**: TR 수십 개
  - 기본 게이트, 플립플롭

**MSI**: TR 수백 개  
  - 디코더, 인코더, MUX, DeMUX, 카운터, 레지스터, 소형기억 장치  

**LSI**: TR 수천개
  - 8비트 마이크로프로세서, 소규모 반도체 기억 장치 칩

**VLSI**: TR 수만~수십만 개
  - 1만 게이트 이상의 논리회로, 단일 칩 마이크로프로세서

**ULSI**: TR 수백만 개

## D4. 폰노이만, 비-폰노이만, 하버드 구조

### 폰 노이만 구조(Von Neumann Architecture)
- 메모리 CPU 간의 연결장치로 구성되는 구조

#### 폰 노이만 구조의 프로그램 처리 과정
- 메모리에서 실행할 명령어A를 인출
- 제어장치(CU)는 명령어A를 해독
- 데이터를 메모리에서 인출 및 레지스터에 저장
- ALU는 명령 실행, 레지스터나 메모리에 결과를 저장

특징: 
- 하나의 기억장소(메모리)에 있음
- 내장 방식 / 축적 프로그램 방식
- CPU와 기억장소가 서로 분리됨
- SIMD(Single Instructure Multiple Data)
- 데이터 버스와 주소 버스를 모두 갖는 현상

#### 폰 노이만 병목 현상
- 데이터를 메모리인출, 저장 반복적 과정 순차적 진행
- 메모리, 시스템 버스에 병목 현상 발생
  - 데이터 경로의 병목 현상, 기억 장소의 지연 현상


### 비 폰노이만 구조
- 정의: 병렬 처리, 추론 기구를 채택한 컴퓨터
- 신경망, 유전 알고리즘, 양자 컴퓨터, 병렬 컴퓨터

### 하버드 구조(Harvard Architecture)
- 프로그램과 데이터가 각각 다르게 분리되어 취급 &rarr; 프로그램 메모리 in Flash Memory, 데이터 메모리 in SRAM
- 프로그램 명령어와 데이터가 다른 비트 수 기반
  - 프로그램 메모리, 데이터 메모리에서 취급
- 특징: 비싸고 공간도 많이 차지, 설계 복잡.
- 임베디드 시스템에서 이러한 형식을 많이 채용함.
  - 아두이노, 매킨토시 등등
