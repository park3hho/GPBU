## H8. 비트와 바이트
### (1) 비트
**정의**: 컴퓨터는 전기의 흐름을 디지털화 하여 0과 1로 표현되는 신호에 의해 동작함
- 컴퓨터에서 표현하는 가장 작은 단위로 0과 1이라는 신호로 표현
### (2) 바이트
**정의**: Byte라는 의미는 컴퓨터에서 한 번에 처리하는 데이터의 크기를 의미함.
- 1byte는 8bit
### (3) Nibble
**정의**: 4bit
### (4) 워드
**정의**: 16bit, 하나의 기계어 명령이나 연산을 통해 저장된 장치로서 **레지스터에 옮겨 놓을 수 있는 데이터 단위.**

# 3주차 2교시: 데이터 형식과 배열
## J10. 숫자 데이터 형식 개요
### (1) 숫자형 데이터 형식 개념
데이터 형식: 선언할 변수의 종류를 의미함
- C언어에서는 변수들은 사용하기 전에 반드시 정수, 실수, 문자열 등의 데이터 형식을 선언해야함

C언어에서 사용되는 숫자 데이터 형식
- 정수형
- 실수형

정수형 데이터 형식  
- **char**: 1byte
- unsigned char: byte
- **short**: 2byte
- unsigned short: 2byte
- **int**: 4byte   
- unsigned int: 4byte
- **long**: 4byte
- unsigned long: 4byte

다양한 자료형이 제공되는 이유
> 데이터의 표현 방식이 다르기 때문임
- C 언어에서는 데이터 형식의 크기를 byte(바이트) 단위로 구하는 [C (B2).md](C%20%28B2%29.md)`sizeof` 연산자가 있음
- CPU가 연산하기에 적합한 데이터의 크기가 int형 크기이다.

#### ㄱ. sizeof 표현방법
```C
int main()
{
  int sizeofNum = 0;
  
  sizeofNum = sizeof sizeofNum;
  sizeofNum = sizeof(int);
  sizeofNum = sizeof(sizeofNum);
}
```


### (2) 실수형 데이터 형식 개념
| 1           | 2 | 3     | 4                |
|-------------|---|-------|------------------|
| float       | B | 4byte | 3.4e-37~3.4e+37  |
| double      | F | 8byte | 1.7e-307~3.4e308 |
| long double | J | 8byte | double 이상의 표현범위  |

- 정수형과 달리 `unsigned`를 표현하지 않음
- 실수의 표현범위
  - flaot, double 둘 다 충분하나 8바이트 크기의 double이 float보다 더 정밀하게 실수 표현 가능

## K11. 문자형 데이터 형식 개념
> 컴퓨터는 문자를 표현 및 저장하지 못하기 때문에 문자를 표현하기 위해 각 문자에 고유한 숫자를 지정함.
- 문자는 따옴표로 표현됨
- 문자를 char에 저장하는 이유
  - 모든 ASCII 코드는 1바이트로도 충분히 표현 가능하기 때문임.

서식 문자 %c를 이용함
- 해당 숫자의 아스키 코드 문자를 출력한 것
```yaml
  char ch1 = 'a'
  printf("%c", ch1)
```

## L12. 리터럴 상수 데이터 형식 개요
정의: 이름을 지니지 않은 상수  
예: int val=30+40;
- 연산을 위해서는 30, 40과 같이 프로그램상에 표현되는 숫자도 메모리 공간에 저장되어야 함.
- 이렇게 저장되는 값은 이름이 존재하지 않고 변경이 불가능한 상수임. 이를 리터럴 상수라고 함.

> 자료형이 결정이 되어야 메모리 공간에 저장될 수 있음
- 정수는 기본적으로 int형
- 실수는 기본적으로 dobule형
- 문자는 기본적으로 char형
```yaml
int main()
{
  char c = 'A';
  int i = 5;
  float f = 3.14;
}
```

### (1) 심볼릭 상수
**정의**: 이름을 지니는 상수
- 심볼릭 상수를 정의하는 방법
  - const 키워드를 사용하고 모두 **대문자**로 표현함
- 둘 이상의 단어를 연결할 때는 언더바를 사용함 (예: `MY_AGE`)

const 키워드에 의한 상수화 예
```
const int MAX = 100;
const double PI= 3.1415;
```

## M13. 데이터 형식의 변환
### (1) 자동형 변환
**정의**: 자동적으로 발생하는 형태의 데이터 형식 변환을 의미함.
- 묵시적 형 변환이라고도 함.
```예시
int n = 5.25 // n=5로 재구성됨.
double d = 245 // 245.0으로 재구성됨
```

- 정수를 실수로 형 변환 할 때는 오차가 발생함
- 큰 정수를 작은 정수로 형 변환 시 작은 정수의 크기에 맞춰 상위 바이트가 소멸됨.

```yaml
int num3 = 129;
char ch = num3; // int형 변수 num3에 저장된 값이 char형으로 자동 변환됨.
```
- 근데 이때 '상위 바이트 손실'이 발생해 `-127`로 도출됨


#### ㄱ. 정수의 승격
- 일반적으로 CPU가 처리하기에 가장 적합한 크기의 정수 자료형을 int로 정의함
- int 형 연산의 속도가 다른 자료형의 연산 속도에 비해서 동일하거나 더 빠름
- short형에서 int형으로 더 커지는 경우를 `정수의 승격`이라고 함.
- 두 피연산자의 자료형은 일치해야함. 일치하지 않으면 자동 형 변환이 일어남.

- **자동 형 변환 규칙**:
char &rarr; int &rarr; long &rarr; longlong &rarr; float &rarr; double &rarr; long double

#### ㄴ. 강제 형 변환
**정의**: 명시적 변환이라고 하며, 프로그래머의 요청에 의한 데이터 형식의 변환을 의미함.
- 변환 형식은 반환하고자 하는 데이터 형식을 `()` 에 넣어 변환함
  - float f = (float)3.14; // 3.14를 float형으로 형 변환
  - dobule e1 = 3 + 3.14; // 정수 3이 double형으로 자동 형 변환
  - double e2 = 3 + (int)3.14 // 3.14가 int형으로 자동 형 변환
